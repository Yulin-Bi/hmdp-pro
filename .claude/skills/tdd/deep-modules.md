# 深度模块

来自《软件设计哲学》：

**深度模块** = 小接口 + 大量实现

```
┌─────────────────────┐
│     小接口           │  ← 方法少、参数简单
├─────────────────────┤
│                     │
│                     │
│   深实现             │  ← 复杂逻辑隐藏在内部
│                     │
│                     │
└─────────────────────┘
```

**浅模块** = 大接口 + 少量实现（应避免）

```
┌─────────────────────────────────┐
│       大接口                    │  ← 方法多、参数复杂
├─────────────────────────────────┤
│  薄实现                         │  ← 仅仅做传递
└─────────────────────────────────┘
```

设计接口时，问自己：

- 能否减少方法数量？
- 能否简化参数？
- 能否将更多复杂性隐藏在内部？
